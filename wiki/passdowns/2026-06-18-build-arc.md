## Gotchas / conventions
- **Worktree constraint:** a branch checked out in one worktree can't be checked out in another. To work
  off someone's branch, branch from `origin/<their-branch>` (e.g. CX integrates via
  `git checkout -b feature/cx-integrate-001 origin/feature/cc-vertical-slice`). Likewise branch new work
  from `origin/main`, never `git checkout main` (repo-master holds it).
- **Integration:** integrator (different model) runs the gate + correctness review, writes a review ending
  `VERDICT: PASS`/`HOLD`; **CM lands on PASS** (`--no-ff` from repo-master). Integrator does not merge.
- **Numbering:** specs on disk: 001 (slice), 003 (GB onboard), 004 (CX integrate). 002 is a gap (the
  CC/CX onboarding *reports* were named spec001/spec002 before the spec-file convention settled). Numbers
  are never reused; just take highest-file + 1.
- Memory lives in the `.claude` dir, not `repo-master/memory` ([[memory-path-guard]]).
